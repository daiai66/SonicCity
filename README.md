# SonicCity

SonicCity 是一个以城市标志性声音为中心的沉浸式声音网站。它收集便利店铃声、地铁 chimes、站内报站、街区白噪声、流行音乐片段等城市声景，让用户通过声音回到旅途中的兴奋、安逸和探索感。

本人是一名旅游爱好者，喜欢到国内外不同城市 citywalk。我发现每座城市都有很多地标性的洗脑声音。每次去过一个城市，大脑总会被这个城市的标志性声音所占据和沉迷，如便利店、地铁 chimes、酒吧音乐等。这些熟悉而悦耳的声音，每每听到总能让人回想起在这座城市的美好回忆与快乐时光。

因此，我希望把这些地标性声音收集起来，做成一个以城市内熟悉声音为中心的网站。这些声音可能是音乐，也可能是白噪声，也可能是到处播放的提示音。对于去过这些地方的人们，它会是一种美好、治愈的声音流脑部按摩，让人放松自己的灵魂，仿佛回到旅途上的那种兴奋、安逸和探索感。即使人们没有去过这些地方，播放这些洗脑的声音也可以培养用户心智，提高这些城市在用户认知中的鲜活度。

这个网站不仅可以选择城市，也引入了 AI 声音推荐引擎，并新增了“状态选择”功能。用户可以根据自己当下的情绪或需求，比如“需要放松”“想专注工作”“有点怀旧”“渴望新鲜感”，让系统从海量城市声景中智能匹配最合适的声音片段。

围绕这些声音所对应的街区、店铺或城市地标，SonicCity 也可以构建一个以“声纹”为介质的新型营销推广平台。商家无需依赖传统图文广告，而是可以将自家产品、手信、美食、品牌形象或线下活动，与所在城市或街区的标志性声音进行深度绑定。例如，听到某条老街的叫卖声，即可弹出附近百年老店的招牌点心链接；听到某座地铁站的经典报站音，即可跳转该站周边的文创好物推荐；听到某条商业街的专属 BGM，即可推送该街区当季的限时优惠或展览信息。

AI 会根据用户的“状态选择”，如“想买伴手礼”“想找当地美食”“想探索小众去处”，以及用户的收听场景，智能匹配最相关的声音-商品组合，实现“听到即种草、心动即链接”的无缝转化。用户在沉浸于声音带来的愉悦与回忆之余，自然接触到有温度、有场景感的本地推荐；商家则获得一条直达精准兴趣用户的情感化推广通道，让广告不再是打扰，而是声音旅程中的自然延伸。

SonicCity 由此在用户与商家之间架起一座以听觉为纽带的桥梁：用户获得治愈与惊喜，商家获得曝光与转化，城市本身则获得更鲜活、更立体的文化传播方式，形成一个多方共赢的声音生态闭环。

## 如何使用

1. 下载或克隆整个仓库。
2. 保持文件结构不变，尤其是 `assets/` 文件夹和根目录下被引用的音频文件。
3. 直接用浏览器打开 `SonicCity.html`。
4. 在页面中选择城市、声音分类或状态需求。
5. 点击音频播放按钮后即可收听。部分浏览器会阻止自动播放声音，所以首次播放通常需要用户主动点击。

如果想通过 GitHub Pages 访问，可以在仓库的 `Settings` -> `Pages` 中选择从 `main` 分支部署，然后访问页面地址中的 `SonicCity.html`。

AI 推荐功能默认不会在公开仓库中包含 API Key。如需启用在线 DeepSeek 推荐，请在本地自行配置密钥，或改造成后端代理调用，避免把密钥写进前端 HTML。

## 文件说明

- `SonicCity.html`：主页面。
- `assets/`：页面图片、地球纹理、Three.js 和音频资源。
- `东京-metro chimes.mp3`、`MRT-黄线.mp3`、`绿线.mp3`、`WCTNY.mp3`：页面直接引用的根目录音频。
- `tools/`：用于压缩或外置资源的辅助脚本。
